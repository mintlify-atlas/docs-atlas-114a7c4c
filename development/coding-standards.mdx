## Naming Conventions

### Files and Directories

- **Components**: Use kebab-case for files (e.g., `add-branch-dialog.tsx`)
- **Pages**: Use `page.tsx` for route pages (Next.js convention)
- **Actions**: Use `actions.ts` for server actions
- **Types**: Use `.ts` extension for type-only files

### Variables and Functions

- **Components**: PascalCase (e.g., `AddBranchDialog`, `BranchSwitcher`)
- **Functions**: camelCase (e.g., `createBranch`, `getAppContext`)
- **Constants**: UPPER_SNAKE_CASE for true constants (e.g., `MAX_ITEMS`)
- **Types/Interfaces**: PascalCase (e.g., `AppContext`, `BranchSummary`)

### Database

- **Tables**: snake_case (e.g., `branch_user`, `user_profile`)
- **Columns**: snake_case (e.g., `full_name`, `base_price`)

## Component Patterns

### Server Actions

All server actions should:
- Use `"use server"` directive
- Be placed in `actions.ts` files
- Accept `FormData` parameters
- Call `revalidatePath` after mutations
- Handle errors appropriately

```typescript
"use server";

import { revalidatePath } from "next/cache";
import { createClient } from "@/utils/supabase/server";

export async function createBranch(formData: FormData) {
  const supabase = await createClient();

  const code = String(formData.get("code") ?? "").trim().toUpperCase();
  const name = String(formData.get("name") ?? "").trim();

  if (!code || !name) {
    throw new Error("Branch code and name are required");
  }

  const { error } = await supabase.from("branch").insert({
    code,
    name,
  });

  if (error) {
    throw error;
  }

  revalidatePath("/branches");
  return { success: true } as const;
}
```

### Client Components

Client components should:
- Use `"use client"` directive at the top
- Utilize React hooks (useState, useTransition, etc.)
- Handle form validation with Zod and React Hook Form
- Display toast notifications for user feedback

```typescript
"use client";

import { useEffect, useState, useTransition } from "react";
import { zodResolver } from "@hookform/resolvers/zod";
import { useForm } from "react-hook-form";
import { z } from "zod";
import { toast } from "sonner";

const schema = z.object({
  code: z.string().min(2, "Code must be at least 2 characters"),
  name: z.string().min(1, "Name is required"),
});

type FormValues = z.infer<typeof schema>;

export function AddBranchDialog() {
  const [open, setOpen] = useState(false);
  const [isPending, startTransition] = useTransition();

  const form = useForm<FormValues>({
    resolver: zodResolver(schema),
    defaultValues: {
      code: "",
      name: "",
    },
  });

  const onSubmit = (values: FormValues) => {
    startTransition(async () => {
      try {
        await createBranch(formData);
        toast.success("Branch created");
        form.reset();
        setOpen(false);
      } catch (error) {
        toast.error(error instanceof Error ? error.message : "Failed");
      }
    });
  };

  return (
    // Component JSX
  );
}
```

## Form Validation

### Zod Schemas

Use Zod for runtime validation:

```typescript
const branchSchema = z.object({
  code: z
    .string()
    .trim()
    .min(2, "Code must be at least 2 characters")
    .max(6, "Code must be at most 6 characters")
    .transform((value) => value.toUpperCase()),
  name: z.string().min(1, "Name is required"),
  email: z
    .string()
    .trim()
    .refine(
      (value) => value === "" || z.string().email().safeParse(value).success,
      { message: "Enter a valid email address" }
    ),
});
```

### React Hook Form

Integrate Zod with React Hook Form:

```typescript
const form = useForm<FormValues>({
  resolver: zodResolver(branchSchema),
  defaultValues: {
    code: "",
    name: "",
  },
});
```

### Query Patterns

```typescript
// Select with joins

  .from("branch_user")
  .select("branch:branch_id (id, code, name)")
  .eq("user_id", user.id)
```

## Best Practices

### Error Handling

- Always handle Supabase errors
- Provide user-friendly error messages
- Use toast notifications for feedback

### Performance

- Use `useTransition` for async operations
- Call `revalidatePath` after mutations
- Leverage Next.js caching strategies

### Security

- Validate all user input with Zod
- Sanitize data before database operations
- Use environment variables for secrets
- Never expose service role keys to the client

### Accessibility

- Use semantic HTML elements
- Include proper ARIA labels
- Ensure keyboard navigation works
- Use Radix UI components for built-in accessibility
