## Overview

Pramukh Men's Wear is built with Next.js 14+ using the App Router architecture. The project follows a feature-based organization with clear separation between authenticated and public routes.

- **`page.tsx`** - UI component for the route

- **Nested routes** - Sub-features organized in subdirectories

## Key Architectural Patterns

### Route Groups

- **`(app)`** - Protected routes requiring authentication. Uses `lib/server/app-context.ts:26` to enforce auth.
- **`(auth)`** - Public authentication pages (login, signup)

Route groups provide layout isolation without affecting URL structure.

### Server Actions Pattern

Each feature module includes an `actions.ts` file with server actions:

```typescript
// Pattern: app/(app)/[feature]/actions.ts
"use server";

import { revalidatePath } from "next/cache";
import { createClient } from "@/utils/supabase/server";

export async function createResource(formData: FormData) {
  // Implementation
}
```

See [Server Actions](/development/server-actions) for detailed patterns.

### Layout Composition

1. **Root Layout** (`app/layout.tsx`) - Theme provider, fonts, metadata
2. **App Layout** (`app/(app)/layout.tsx`) - Sidebar, header, navigation, auth context
3. **Page** - Feature-specific UI

The app layout uses `getAppContext()` from `lib/server/app-context.ts:26` to load user and branch data.

### Component Naming Conventions

- **Dialogs**: `add-[resource]-dialog.tsx` - Modal forms for creating resources
- **Menus**: `[resource]-action-menu.tsx` - Dropdown menus for resource actions
- **Forms**: Prefixed with `form-` for form-specific components
- **Pages**: Always `page.tsx` (Next.js convention)
- **Actions**: Always `actions.ts` (project convention)

## File Naming

- **React components**: `kebab-case.tsx`
- **Server actions**: `actions.ts`
- **Types**: `kebab-case.ts`
- **Utilities**: `kebab-case.ts`

## Import Aliases

The project uses TypeScript path aliases:

```typescript
import { Button } from "@/components/ui/button";
import { getAppContext } from "@/lib/server/app-context";
import { createClient } from "@/utils/supabase/server";
```

`@/` maps to the project root directory.

## Database Integration

Database interactions use Supabase clients:

- **Server components/actions**: `createClient()` from `@/utils/supabase/server`
- **Client components**: `createClient()` from `@/utils/supabase/client`
- **Middleware**: `createClient()` from `@/utils/supabase/middleware`
