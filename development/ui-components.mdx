## Overview

Pramukh Men's Wear uses [shadcn/ui](https://ui.shadcn.com/) for its component library, built on top of Radix UI primitives and styled with Tailwind CSS. The project follows the "new-york" style preset with custom theming.

## Configuration

The shadcn/ui configuration is defined in `components.json`:

```json
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "new-york",
  "rsc": true,
  "tsx": true,
  "tailwind": {
    "config": "",
    "css": "app/globals.css",
    "baseColor": "neutral",
    "cssVariables": true,
    "prefix": ""
  },
  "iconLibrary": "lucide",
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui",
    "lib": "@/lib",
    "hooks": "@/hooks"
  }
}
```

### Key Configuration Details

- **Style**: `new-york` - Modern component design with cleaner aesthetics
- **RSC**: React Server Components enabled
- **Base Color**: `neutral` - Provides the color palette foundation
- **CSS Variables**: Enabled for dynamic theming
- **Icon Library**: Lucide React for consistent iconography

## Core UI Components

All UI components are located in `components/ui/` and follow Radix UI patterns.

### Button Component

The button component uses `class-variance-authority` for variant management:

```typescript
import { Button } from "@/components/ui/button";

// Usage examples
<Button>Default</Button>
<Button variant="destructive">Delete</Button>
<Button variant="outline">Cancel</Button>
<Button variant="ghost">Ghost</Button>
<Button size="sm">Small</Button>
<Button size="lg">Large</Button>
<Button size="icon">⚙️</Button>
```

**Available Variants:**
- `default` - Primary action button
- `destructive` - Dangerous actions
- `outline` - Secondary actions
- `secondary` - Alternative actions
- `ghost` - Minimal styling
- `link` - Text link styling

**Available Sizes:**
- `default` - h-9 with standard padding
- `sm` - h-8 smaller button
- `lg` - h-10 larger button
- `icon` - h-9 w-9 square button

## Custom Application Components

Custom components in `components/app/` extend the base UI components with application-specific logic.

### FormSubmitButton

Button that automatically shows loading state during form submission:

```typescript
import { FormSubmitButton } from "@/components/app/form-submit-button";

<form action={serverAction}>
  {/* Form fields */}
  <FormSubmitButton pendingLabel="Saving...">
    Save Changes
  </FormSubmitButton>
</form>
```

**Implementation:**

```typescript
"use client";

import { useFormStatus } from "react-dom";
import { Button, type ButtonProps } from "@/components/ui/button";

type FormSubmitButtonProps = {
  pendingLabel?: string;
} & ButtonProps;

export function FormSubmitButton({
  children,
  pendingLabel = "Saving...",
  disabled,
  ...props
}: FormSubmitButtonProps) {
  const { pending } = useFormStatus();

  return (
    <Button type="submit" disabled={disabled ?? pending} {...props}>
      {pending ? pendingLabel : children}
    </Button>
  );
}
```

### ModeToggle

Theme switcher using next-themes:

```typescript
import { ModeToggle } from "@/components/app/mode-toggle";

<ModeToggle />
```

**Implementation:**

```typescript
"use client";

import { MoonIcon, SunIcon } from "lucide-react";
import { useTheme } from "next-themes";
import { Toggle } from "@/components/ui/toggle";

export function ModeToggle() {
  const { theme, setTheme } = useTheme();
  const isDark = theme === "dark";

  return (
    <Toggle
      aria-label="Toggle theme"
      pressed={isDark}
      onPressedChange={(pressed) => setTheme(pressed ? "dark" : "light")}
    >
      <SunIcon className="size-4 rotate-0 scale-100 transition-all dark:-rotate-90 dark:scale-0" />
      <MoonIcon className="absolute size-4 rotate-90 scale-0 transition-all dark:rotate-0 dark:scale-100" />
    </Toggle>
  );
}
```

### SidebarNav

Navigation component with active state styling:

```typescript
import { SidebarNav } from "@/components/app/sidebar-nav";

const navItems = [
  { href: "/dashboard", label: "Dashboard" },
  { href: "/vendors", label: "Vendors" },
  { href: "/purchases", label: "Purchases" },
];

<SidebarNav items={navItems} />
```

**Implementation:**

```typescript
"use client";

import Link from "next/link";
import { usePathname } from "next/navigation";

export type NavItem = {
  href: string;
  label: string;
};

export function SidebarNav({ items }: { items: NavItem[] }) {
  const pathname = usePathname();

  return (
    <nav className="space-y-2">
      {items.map((item) => {
        const isActive = pathname === item.href;
        return (
          <Link
            key={item.href}
            href={item.href}
            className={`block rounded-2xl px-4 py-3 text-sm font-semibold uppercase tracking-[0.28em] transition ${
              isActive
                ? "bg-primary/20 text-primary"
                : "text-white/60 hover:bg-white/10 hover:text-white"
            }`}
          >
            {item.label}
          </Link>
        );
      })}
    </nav>
  );
}
```

## Best Practices

### Component Composition

Compose existing components rather than creating new ones:

```typescript
// Good: Compose existing components
import { Button } from "@/components/ui/button";
import { Dialog, DialogContent, DialogTrigger } from "@/components/ui/dialog";

function MyDialog() {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button>Open</Button>
      </DialogTrigger>
      <DialogContent>
        {/* Custom content */}
      </DialogContent>
    </Dialog>
  );
}
```

### Server vs Client Components

Use `"use client"` directive only when necessary:

```typescript
// UI components that need interactivity
"use client";

import { useState } from "react";
import { Button } from "@/components/ui/button";

export function InteractiveComponent() {
  const [count, setCount] = useState(0);
  return <Button onClick={() => setCount(count + 1)}>{count}</Button>;
}
```

### Styling Guidelines

1. Use Tailwind utility classes
2. Leverage CSS variables for theming
3. Use `cn()` utility for conditional classes
4. Maintain consistent spacing with Tailwind spacing scale

```typescript
import { cn } from "@/lib/utils";

<div className={cn(
  "base-classes",
  isActive && "active-classes",
  className // Allow override
)} />
```

## Adding New Components

To add shadcn/ui components:

```bash
npx shadcn@latest add [component-name]
```

Example:

```bash
npx shadcn@latest add dropdown-menu
npx shadcn@latest add popover
```

This automatically installs the component with proper configuration and dependencies.
