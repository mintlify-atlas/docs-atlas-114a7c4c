This guide covers all environment variables needed to run Pramukh Men's Wear. These variables configure the connection to your Supabase project and enable authentication and database operations.

## Required Variables

The following environment variables must be set for the application to function properly.

<ParamField path="NEXT_PUBLIC_SUPABASE_URL" type="string" required>
  Your Supabase project URL. Find this in your Supabase project dashboard under Settings > API.
  
  Example: `https://your-project.supabase.co`
</ParamField>

<ParamField path="NEXT_PUBLIC_SUPABASE_ANON_KEY" type="string" required>
  The Supabase anonymous (public) key. Also called the publishable key. This is safe to use in client-side code.
  
  Find this in your Supabase project dashboard under Settings > API > Project API keys > anon/public.
  
  <Note>
    The application also accepts `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` as an alias for this variable.
  </Note>
</ParamField>

## Optional Variables

<ParamField path="SUPABASE_SERVICE_ROLE_KEY" type="string">
  The Supabase service role key with elevated privileges. Required for:
  - Running database migrations via the Supabase CLI
  - Administrative operations that bypass Row Level Security
  
  Find this in your Supabase project dashboard under Settings > API > Project API keys > service_role.
  
  <Warning>
    This key has full access to your database. Keep it secure and never expose it in client-side code. Only use it in server-side operations or local development.
  </Warning>
</ParamField>

## Setting Up Environment Variables

<Steps>
  <Step title="Create environment file">
    Copy the environment template to create your local configuration:
    
    ```bash
    cp .env .env.local
    ```
  </Step>
  
  <Step title="Get Supabase credentials">
    1. Go to your [Supabase dashboard](https://supabase.com/dashboard)
    2. Select your project
    3. Navigate to Settings > API
    4. Copy the Project URL and anon key
  </Step>
  
  <Step title="Add credentials to .env.local">
    Open `.env.local` and add your credentials:
    
    ```bash
    NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
    SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here
    ```
  </Step>
  
  <Step title="Verify configuration">
    Start the development server to verify your environment variables are correctly configured:
    
    ```bash
    pnpm dev
    ```
    
    If configured correctly, you should be redirected to the login page at `http://localhost:3000/login`.
  </Step>
</Steps>

## Environment Variable Usage

The application uses these environment variables in different contexts:

### Client-Side (Browser)

Variables prefixed with `NEXT_PUBLIC_` are available in the browser:

- `NEXT_PUBLIC_SUPABASE_URL` - Used by the Supabase client to connect to your project
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` - Used for authentication and authorized database queries

These are used in:
- `/utils/supabase/client.ts` - Browser client creation
- `/utils/supabase/server.ts` - Server-side client creation
- `/utils/supabase/middleware.ts` - Authentication middleware

### Server-Side Only

- `SUPABASE_SERVICE_ROLE_KEY` - Only used for migrations and administrative tasks

<Note>
  The application checks for both `NEXT_PUBLIC_SUPABASE_ANON_KEY` and `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` and will use whichever is available. You only need to set one of these.
</Note>

## Production Deployment

For production deployments (Vercel, Netlify, etc.):

1. Add environment variables in your hosting platform's dashboard
2. Do not commit `.env.local` or any file containing real credentials to version control
3. Only deploy the service role key if needed for automated migrations

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error: Missing NEXT_PUBLIC_SUPABASE_URL">
    This error occurs when the Supabase URL is not set. Verify that:
    - Your `.env.local` file exists
    - The variable name is exactly `NEXT_PUBLIC_SUPABASE_URL`
    - There are no extra spaces around the `=` sign
    - You restarted the dev server after adding the variable
  </Accordion>
  
  <Accordion title="Error: Missing NEXT_PUBLIC_SUPABASE_ANON_KEY">
    This error occurs when the anon key is not set. Check that:
    - You've set either `NEXT_PUBLIC_SUPABASE_ANON_KEY` or `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`
    - The key is copied correctly from Supabase dashboard
    - You restarted the dev server
  </Accordion>
  
  <Accordion title="Authentication not working">
    If authentication fails:
    - Verify your Supabase URL and anon key are correct
    - Check that your Supabase project is active
    - Ensure the keys are from the same Supabase project
    - Check the browser console for detailed error messages
  </Accordion>
</AccordionGroup>
