## Overview

This module manages the procurement side of the business: vendor relationships, purchase orders with line items, and vendor payment tracking.

## Tables

### vendor

Stores information about suppliers and vendors.

```sql
create table public.vendor (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  contact_name text,
  phone text,
  email text,
  gst_number text,
  address text,
  created_at timestamptz not null default timezone('utc', now()),
  updated_at timestamptz not null default timezone('utc', now())
);
create unique index vendor_name_key on public.vendor (name);
create unique index vendor_phone_key on public.vendor (phone) where phone is not null;
```

- **Unique index**: `name` must be unique
- **Conditional unique index**: `phone` is unique when not null
- **Trigger**: `set_timestamp_vendor` - automatically updates `updated_at`

#### Relationships

- **purchase** (1:N) - Vendor can have multiple purchase orders
- **vendor_payment** (1:N) - Vendor can receive multiple payments

### purchase

Records a purchase order/invoice from a vendor.

```sql
create table public.purchase (
  id uuid primary key default gen_random_uuid(),
  branch_id uuid not null references public.branch(id) on delete restrict,
  vendor_id uuid not null references public.vendor(id) on delete restrict,
  ordered_by uuid references public.user_profile(id) on delete set null,
  invoice_number text,
  purchase_date date not null default current_date,
  subtotal numeric(12, 2) not null default 0,
  discount_total numeric(12, 2) not null default 0,
  tax_total numeric(12, 2) not null default 0,
  total_amount numeric(12, 2) not null default 0,
  notes text,
  created_at timestamptz not null default timezone('utc', now()),
  updated_at timestamptz not null default timezone('utc', now())
);
create unique index purchase_invoice_vendor_unique on public.purchase (vendor_id, invoice_number) where invoice_number is not null;
```

#### Columns

<ResponseField name="id" type="uuid" required>
  Primary key, auto-generated UUID
</ResponseField>

<ResponseField name="branch_id" type="uuid" required>
  Reference to branch receiving the goods. Prevents deletion.
</ResponseField>

<ResponseField name="vendor_id" type="uuid" required>
  Reference to vendor supplying the goods. Prevents deletion.
</ResponseField>

<ResponseField name="ordered_by" type="uuid">
  Reference to user who created the purchase order. Set to null if user is deleted.
</ResponseField>

<ResponseField name="invoice_number" type="text">
  Vendor's invoice number (unique per vendor)
</ResponseField>

<ResponseField name="purchase_date" type="date" required default="current_date">
  Date of purchase/invoice
</ResponseField>

<ResponseField name="subtotal" type="numeric(12,2)" required default="0">
  Sum of all line items before discounts and taxes
</ResponseField>

<ResponseField name="discount_total" type="numeric(12,2)" required default="0">
  Total discount amount applied
</ResponseField>

<ResponseField name="tax_total" type="numeric(12,2)" required default="0">
  Total tax amount (GST, VAT, etc.)
</ResponseField>

<ResponseField name="total_amount" type="numeric(12,2)" required default="0">
  Final amount: subtotal - discount_total + tax_total
</ResponseField>

<ResponseField name="notes" type="text">
  Additional notes or comments
</ResponseField>

<ResponseField name="created_at" type="timestamptz" required>
  Record creation timestamp (UTC)
</ResponseField>

<ResponseField name="updated_at" type="timestamptz" required>
  Last update timestamp (UTC), automatically updated by trigger
</ResponseField>

#### Constraints & Indexes

- **Foreign key**: `branch_id` references `branch(id)` with `ON DELETE RESTRICT`
- **Foreign key**: `vendor_id` references `vendor(id)` with `ON DELETE RESTRICT`
- **Foreign key**: `ordered_by` references `user_profile(id)` with `ON DELETE SET NULL`
- **Composite unique index**: `(vendor_id, invoice_number)` when invoice_number is not null
- **Trigger**: `set_timestamp_purchase` - automatically updates `updated_at`

#### Relationships

- **branch** (N:1) - Purchase belongs to a branch
- **vendor** (N:1) - Purchase is from a vendor
- **user_profile** (N:1) - Purchase created by a user
- **purchase_item** (1:N) - Purchase has multiple line items
- **vendor_payment** (1:N) - Purchase can have multiple payments

### purchase_item

Individual line items in a purchase order.

```sql
create table public.purchase_item (
  id uuid primary key default gen_random_uuid(),
  purchase_id uuid not null references public.purchase(id) on delete cascade,
  item_variant_id uuid not null references public.item_variant(id) on delete restrict,
  quantity numeric(12, 2) not null check (quantity > 0),
  unit_cost numeric(12, 2) not null check (unit_cost >= 0),
  discount numeric(12, 2) not null default 0,
  total_cost numeric(12, 2) not null default 0,
  created_at timestamptz not null default timezone('utc', now())
);
create unique index purchase_item_unique_variant_per_purchase on public.purchase_item (purchase_id, item_variant_id);
```

#### Columns

<ResponseField name="id" type="uuid" required>
  Primary key, auto-generated UUID
</ResponseField>

<ResponseField name="purchase_id" type="uuid" required>
  Reference to parent purchase. Cascade deletes when purchase is deleted.
</ResponseField>

<ResponseField name="item_variant_id" type="uuid" required>
  Reference to item variant being purchased. Prevents deletion.
</ResponseField>

<ResponseField name="quantity" type="numeric(12,2)" required>
  Quantity purchased (must be > 0)
</ResponseField>

<ResponseField name="unit_cost" type="numeric(12,2)" required>
  Purchase cost per unit (must be >= 0)
</ResponseField>

<ResponseField name="discount" type="numeric(12,2)" required default="0">
  Discount amount per unit
</ResponseField>

<ResponseField name="total_cost" type="numeric(12,2)" required default="0">
  Line total: (unit_cost - discount) * quantity
</ResponseField>

<ResponseField name="created_at" type="timestamptz" required>
  Record creation timestamp (UTC)
</ResponseField>

#### Constraints & Indexes

- **Foreign key**: `purchase_id` references `purchase(id)` with `ON DELETE CASCADE`
- **Foreign key**: `item_variant_id` references `item_variant(id)` with `ON DELETE RESTRICT`
- **Check constraint**: `quantity > 0`
- **Check constraint**: `unit_cost >= 0`
- **Composite unique index**: `(purchase_id, item_variant_id)` - prevents duplicate variants in same purchase

#### Relationships

- **purchase** (N:1) - Line item belongs to a purchase
- **item_variant** (N:1) - Line item is for a specific variant

### vendor_payment

Tracks payments made to vendors, linked to specific purchases or general account payments.

```sql
create table public.vendor_payment (
  id uuid primary key default gen_random_uuid(),
  vendor_id uuid not null references public.vendor(id) on delete cascade,
  purchase_id uuid references public.purchase(id) on delete set null,
  amount numeric(12, 2) not null check (amount > 0),
  method public.payment_method not null default 'cash',
  paid_at timestamptz not null default timezone('utc', now()),
  reference text,
  notes text,
  created_at timestamptz not null default timezone('utc', now())
);
create index vendor_payment_vendor_idx on public.vendor_payment (vendor_id);
create index vendor_payment_purchase_idx on public.vendor_payment (purchase_id);
```

#### Columns

<ResponseField name="id" type="uuid" required>
  Primary key, auto-generated UUID
</ResponseField>

<ResponseField name="vendor_id" type="uuid" required>
  Reference to vendor. Cascade deletes when vendor is deleted.
</ResponseField>

<ResponseField name="purchase_id" type="uuid">
  Optional reference to specific purchase. Set to null if purchase is deleted.
</ResponseField>

<ResponseField name="amount" type="numeric(12,2)" required>
  Payment amount (must be > 0)
</ResponseField>

<ResponseField name="method" type="payment_method" required default="cash">
  Payment method used.
  
  **Enum values**: `cash`, `upi`, `card`, `other`
</ResponseField>

<ResponseField name="paid_at" type="timestamptz" required default="now()">
  Date and time when payment was made
</ResponseField>

<ResponseField name="reference" type="text">
  Reference number (transaction ID, check number, etc.)
</ResponseField>

<ResponseField name="notes" type="text">
  Additional notes about the payment
</ResponseField>

<ResponseField name="created_at" type="timestamptz" required>
  Record creation timestamp (UTC)
</ResponseField>

#### Constraints & Indexes

- **Foreign key**: `vendor_id` references `vendor(id)` with `ON DELETE CASCADE`
- **Foreign key**: `purchase_id` references `purchase(id)` with `ON DELETE SET NULL`
- **Check constraint**: `amount > 0`
- **Index**: `vendor_payment_vendor_idx` on `vendor_id`
- **Index**: `vendor_payment_purchase_idx` on `purchase_id`

## Transaction Flow

### Create Purchase Order

<Steps>
  <Step title="Create Purchase">
    Insert into `purchase` table with vendor, branch, and invoice details
  </Step>
  <Step title="Add Line Items">
    Insert into `purchase_item` for each product being purchased
  </Step>
  <Step title="Calculate Totals">
    Update `purchase` with subtotal, discounts, taxes, and total_amount
  </Step>
  <Step title="Record Payment (if immediate)">
    Insert into `vendor_payment` linked to the purchase
  </Step>
</Steps>

### Manage Vendor Payments

<Steps>
  <Step title="Track Outstanding Amount">
    Calculate: `purchase.total_amount` - sum of related `vendor_payment.amount`
  </Step>
  <Step title="Make Partial Payment">
    Insert into `vendor_payment` with specific purchase reference
  </Step>
  <Step title="Make Account Payment">
    Insert into `vendor_payment` without purchase reference (general account credit)
  </Step>
</Steps>

## Usage Examples

### Create Vendor

```sql
insert into public.vendor (name, contact_name, phone, email, gst_number, address)
values (
  'Premium Fabrics Ltd',
  'Rajesh Kumar',
  '+91-9876543210',
  'sales@premiumfabrics.com',
  '29ABCDE1234F1Z5',
  '123 Textile Market, Mumbai, Maharashtra 400001'
);
```

### Create Purchase Order

```sql
-- Create purchase header
insert into public.purchase (branch_id, vendor_id, ordered_by, invoice_number, purchase_date)
values (
  '...branch-uuid...',
  '...vendor-uuid...',
  '...user-uuid...',
  'INV-2024-001',
  '2024-03-15'
);

-- Add line items
insert into public.purchase_item (purchase_id, item_variant_id, quantity, unit_cost, total_cost)
values 
  ('...purchase-uuid...', '...variant-1-uuid...', 50, 500.00, 25000.00),
  ('...purchase-uuid...', '...variant-2-uuid...', 30, 600.00, 18000.00);

-- Update purchase totals
update public.purchase
set 
  subtotal = 43000.00,
  tax_total = 7740.00,  -- 18% GST
  total_amount = 50740.00
where id = '...purchase-uuid...';
```

### Record Vendor Payment

```sql
-- Full payment for a specific purchase
insert into public.vendor_payment (vendor_id, purchase_id, amount, method, reference)
values (
  '...vendor-uuid...',
  '...purchase-uuid...',
  50740.00,
  'upi',
  'TXN123456789'
);

-- Partial payment
insert into public.vendor_payment (vendor_id, purchase_id, amount, method, notes)
values (
  '...vendor-uuid...',
  '...purchase-uuid...',
  25000.00,
  'cash',
  'Partial payment - balance to be paid next month'
);

-- Account credit (not for specific purchase)
insert into public.vendor_payment (vendor_id, amount, method, notes)
values (
  '...vendor-uuid...',
  10000.00,
  'cash',
  'Advance payment for future orders'
);
```

## Vendor Account Balance

To calculate outstanding amount for a vendor:

```sql
select 
  v.name as vendor_name,
  coalesce(sum(p.total_amount), 0) as total_purchases,
  coalesce(sum(vp.amount), 0) as total_payments,
  coalesce(sum(p.total_amount), 0) - coalesce(sum(vp.amount), 0) as outstanding_balance
from public.vendor v
left join public.purchase p on p.vendor_id = v.id
left join public.vendor_payment vp on vp.vendor_id = v.id
where v.id = '...vendor-uuid...'
group by v.id, v.name;
```

## Business Rules

### Invoice Uniqueness

Each vendor can have multiple purchases, but invoice numbers must be unique per vendor:

```sql
-- This constraint ensures invoice numbers don't duplicate per vendor
create unique index purchase_invoice_vendor_unique 
on public.purchase (vendor_id, invoice_number) 
where invoice_number is not null;
```

### Variant Uniqueness in Purchase

A purchase order cannot have the same variant listed twice (prevent duplicate line items):

```sql
-- Ensures each variant appears only once per purchase
create unique index purchase_item_unique_variant_per_purchase 
on public.purchase_item (purchase_id, item_variant_id);
```

### Data Protection

Both `branch_id` and `vendor_id` use `ON DELETE RESTRICT` to prevent accidental deletion:

- Cannot delete a branch that has purchases
- Cannot delete a vendor that has purchases
- Cannot delete an item variant that appears in purchase items

This ensures historical purchase data integrity.
